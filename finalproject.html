Anna's Final Project: Medford!

This is my final Project! I expanded on a prototype I made in 451 to help remind me to take my medicine. I made a little friend named Medford who will be disapointed if I don't take my meds, and tells me when the last time I took each of my doses is.

Schematic

Shown on the schematic are 15 Arduino pins, 8 components and 5 resistors. The 5V pin is connected to a breadboard and supplies power for the LCD, RTC, and LED Matrix. The LED matrix is connected to the 5V pin for power, the 8 pin for data input, and the ground pin. The LCD screen is connected to the 5V pin at various points for power, pins 2,3,4,5, 11, and 12 for various data inputs, and is connected to ground at various points. It is also connected to a potentiometer, which controls the LCD display brightness. The potentiometer is connected to 5V and ground also. The RTC module is connected to the 5V pin for power, the SCL pin, the SDA pin, and the ground pin.

Pin 7 is connected to and provides power to two red LEDs in series, which are then connected to two 220 ohm resistors and ground. Pin 6 is connected to two photoresists in series, which are then connected to pins A0 and A1, respectively, and are then connected to two 10k ohm resistors and then ground.

The schematic also shows the values of the resistors, which I determined by using V=IR and using the resistors I had that matched closest. For the red portion of the RGB LED, as their voltage drop is 1.8V, the voltage that is being supplied is 5V, and the target current for LEDs is 20 mA, the resistor needed to reach that target current have to have a resistance of 160 ohms. As I only had 100 and 220 ohm resistors, I opted for 220 ohms as 100ohms would exceed the recommended current maximum of 30mA. For the photoresistors, I used a 10k ohm resistor on the other side of the voltage divider. This was based on the Adafruit Photocells informational document. As it described that higher resistance resistors would allow the photoresistor to be more sensitive to changes in low light, I decided to use a 10k ohm resistor.

My sketch includes libraries, defines variables before the set up method, a set up method, and a loop method. The variables and libraries defined prior to set up fall into three categories: LED Matrix, Photoresistor, and RTC/LCD. The LED Matrix libraries include avr/pgmspace to store things in flash memory and FastLED to control the LEDs. The variables include NUM_LEDS that defines the number of LEDs in the matrix, DATA_PIN which defines the pin used for data input, leds[NUM_LEDS] which defines an array that corresponds to LEDS in the matrix, and 8 arrays that contain the color information for the displays on the LED matrix. The photo resistor includes variables that indicate to the Arduino to check for a change in lighting, variables that track the sensor values, their mins, and their maxes, variables that represent the analog input pins reading their values and the pins that give the LEDs and photoresistsors power, counters that track how long a light change has occurred for, and variables that keep track of the status (defined as light and dark) of the medicine holders. The RTC/LCD portion includes the RTClib and LiquidCrystal libraries, then creates an etc object, defines an array that tells the days of the week, then defines two datetime objects for time tracking, and creates a liquid crystal object.

The loop method first captures the current time, then out checks if the current time is between 10pm and 12am. If it is, it will check for if the night medicine was last taken in the same day as the current day, and will display accordingly. If its between 7am and 12pm, it will check if the morning medicine was last taken on the same day as the current day and display accordingly. Then it will start checking the photoresists readings. If the photoresistor reads above or below a the value of 40, (which I tested to be a pretty good indication of medicine bottle coverage), It will start the counter for light or dark respectively, and after the counter reaches 4 (about 5 seconds), it will define the status of that photoresistor and medicine holder to “dark” or “light”, and indicate that it is time to check the status of both the medicine holders together. If one sensor reads as dark and the other as light (meaning the medicine bottle is in one of the holders), then it will present that time on the LCD screen and record it as the last time one medicine dose was taken.
